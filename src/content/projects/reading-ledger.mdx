Reading Ledger is a small experiment in keeping book notes connected to the act of reading instead of storing them in a separate graveyard. The shape is intentionally modest: books, sessions, notes, and a few views that make neglected reading visible.

## Constraint

The project only works if entering data is cheaper than avoiding it. That means the core flow needs to accept partial information, keep forms short, and make the next action obvious.

## What I Learned

The useful lesson was that personal tools need fewer abstractions than public tools. A private system can keep the model close to the way one person thinks. That makes the code smaller and the interface more honest.
