Local-first software has a different emotional texture. It makes the user's machine feel capable again. The interesting design problem is not only sync; it is trust.

I keep connecting this to [[quiet-abstractions]]. The best local-first tools make distributed systems feel ordinary without hiding the fact that hard work is happening underneath.
